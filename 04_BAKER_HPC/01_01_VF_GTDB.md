# Virulence-factor profiling with DIAMOND

This is a resumed description of the pipeline we used to profile virulence factors in bacterial genomes from GTDB r214, using DIAMOND to align genome sequences against the VFDB database.

The details of the pipeline are as follows:
- Genomes: 394,913 from Bacteria domain
- Source: genomes listed on GTDB r214 metadata (`bac120_metadata_r214.tsv`) and downloadable with NCBI datasets tool (example workflow: https://gazollavolpiano.github.io/bacterial-archaeal-genomes-download.html)
- Virulence factors database: `VFDB_setA_pro.dmnd` (downloaded 11 April 2023), from VFDB
- Query type: nucleotide genomes -> use blastx (translates DNA to protein)
- DIAMOND version: v2.1.6.160

One-liner for a single genome is provided below.

```bash
# DIAMOND command example for a single genome
diamond blastx \
  --db VFDB_setA_pro.dmnd \
  --query genome.fna \
  --out genome_diamond.txt \
  --outfmt 6 qtitle stitle pident length qstart qend sstart send evalue bitscore qcovhsp scovhsp slen \
  # per query: report up to 5000 distinct VFDB proteins (the entire database) and keep the best-scoring alignment per protein
  --max-target-seqs 5000 --max-hsps 1 \  
  # minimum percent identity to report is 50%
  --id 50 \                              
  --threads 8
```

# Provided files
- `gtdb_r124_vfdb_setA_diamond_70ident_90scov_filtered_hits.Rdata`: master table after being filtered, contains 334,408 genomes with at least one hit passing filtering criteria (70% identity and 90% gene coverage)
- `list_394913_gtdb_r124_genomes_included.txt`: a list with the 394,913 genomes included in the analysis
- `gtdb_r124_vfdb_setA_diamond_unfiltered_csv_files.tar.zst`: compressed folder with unfiltered raw DIAMOND `.csv` hits files, it contains 394,904 files (9 genomes had no hits at all)
- `diamond_csv_unfiltered_files.sha256`: checksum file to verify integrity of the extracted files
- `pipeline_to_share.md`: this file, describing the pipeline used

The unfiltered raw DIAMOND `.csv` hits files contain the following columns:
- ncbi_assembly_accession:  NCBI assembly accession
- gtdb_taxonomy: concatenated GTDB taxonomy. e.g. d__Bacteria;p__Pseudomonadota;... (from GTDB `bac120_metadata_r214.tsv` metadata file)
- vfid: VFDB unique ID. e.g. VF0487
- vf_name: virulence factor name. e.g. Rvh T4SS
- vf_category: VF category/group. e.g. Effector delivery system
- reference_species_vfdb_metadata: reference strain/species in VFDB metadata title. e.g. Escherichia coli
- subject_title: VFDB subject line (DIAMOND sequence title)
- aln_len: alignment length
- slen: subject (VF) sequence length used for coverage calcs
- pident: percent identity
- evalue: alignment E-value
- bitscore: alignment bit score
- coverage_alnlen: gene coverage, calculated as (aln_len / slen) * 100
- pass_id_70 (bool): pident >= 70 
- pass_cov_90 (bool): whether coverage_alnlen ≥ 90

To decompress the .tar.zst file use `tar --use-compress-program=unzstd -xvf gtdb_r124_vfdb_setA_diamond_unfiltered_csv_files.tar.zst`. To verify files after extracting: `(cd diamond_csv_unfiltered_files && sha256sum -c ../diamond_csv_unfiltered_files.sha256)`
